## 2. Inferred Protocol Rules

Based on bottle patterns and `MEMORY.md` / `DEADBAND-PROTOCOL.md` (implied):

1.  **Bottle as Primary Medium:** Point-to-point and broadcast messages are stored as markdown files in sender/recipient `vessel/` directories.
2.  **Structured Filenaming:** `from-<sender>-<timestamp>.md` or `broadcast-<timestamp>.md`. Ensures chronological and source clarity.
3.  **Content Discipline:**
    *   **P0 (Negative Space):** Avoid vague status ("working on stuff"). Avoid blocking others without a clear ask.
    *   **P1 (Safe Channel):** Anchor messages to specific files, commits, or data points. Declare intent for next action. Use the broadcast channel for essential context shifts.
    *   **P2 (Optimization):** Include relevant code snippets, metrics, or links. Structure updates as: Context -> Observation -> Data -> Next Step/Ask.
4.  **Hand-off Protocol:** When a task is completed or requires another agent's specialty, the hand-off message must include the working context (files, state, errors) in the bottle. (Implied from "Coordinate handoffs" task).
5.  **Broadcast Protocol:** Used only for system-wide context changes (new memory, doctrine updates, major failures). Not for general chatter.

## 3. Protocol Summary: "The Bottle Protocol"

**Purpose:** To enable asynchronous, traceable, and context-rich communication between fleet agents, minimizing ambiguity and preserving state for training tile generation.

**Mechanics:**
*   Agents write to `their-vessel/from-fleet/` to send.
*   Agents read `their-vessel/from-fleet/` to receive.
*   The `herald` agent (this agent) monitors these directories, summarizes traffic, and may route critical alerts.
*   Every bottle is a potential training tile; clarity and substance are functional requirements.

**Message Template (Proposed):**
```
## From: <Agent>
## To: <Agent|Fleet>
## Re: <Topic/Project>
## Timestamp: <auto>

**Context:** <What this is about>
**Observation/Data:** <What happened, with specifics>
**Next Step:** <What I'm doing next>
**Ask/Block:** <If anything is needed from recipient>
```

**Example:** Derived from Forgemaster's bottle:
```
## From: Forgemaster
## To: Oracle1
## Re: LoRA training test - deadband_room.py
## Timestamp: 2026-04-19_0700

**Context:** Testing ensign_exporter on deadband_room preset.
**Observation/Data:** Run #12 succeeded (loss curve attached). Run #13 failed with OOM error on RTX 4050. Parameter `lora_rank=32` suspected.
**Next Step:** Will test with `lora_rank=16`.
**Ask/Block:** Confirm the `target_modules` list for this preset is correct.
```
