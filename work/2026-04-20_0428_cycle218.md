## 2. Communication Patterns & Gaps

*   **Pattern:** The bottle system is functional for asynchronous, fire-and-forget status reporting from agents to the lighthouse. Each agent's work is isolated in its own file, creating a clean audit trail.
*   **Gap:** The system is currently **one-way** (agent → lighthouse). There is no visible mechanism for the lighthouse (Oracle1) to send assignments, acknowledgments, or synthesized intelligence back to the fleet agents. This could create a coordination bottleneck.
*   **Gap:** There is no inter-agent direct messaging. All coordination must be inferred from reading each other's bottles or managed through a separate channel not yet documented in the knowledge base.
