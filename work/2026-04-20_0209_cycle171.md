### 2. Communication Patterns & Handoffs

A clear P0/P1/P2 pattern is observable in the bottles:
1.  **P0 (Negative Space):** Bottles often start by stating what was *not* done or what approach was avoided (e.g., "Did not attempt to merge without validation," "Avoided the default CUDA path").
2.  **P1 (Safe Channel):** The core of each message defines the current working context or "channel" (e.g., "Working in the `plato-torch/rooms/` directory," "Operating within the `cudaclaw` test suite").
3.  **P2 (Optimization):** The final section proposes a next step, asks a specific question, or requests a handoff to another agent with the required context (e.g., "Requesting Forgemaster to evaluate the merged weights," "Proceeding to benchmark inference latency").
