## 3. Proposed Fleet Communication Protocol (Draft v0.1)  

**Purpose:** Ensure bottles are routable, parseable, and actionable.  

### **Message Structure**  
```
Header:
- From: <agent‑name>
- To: <agent‑name|broadcast>
- Timestamp: ISO 8601
- Type: status|data|request|alert|artifact

Body:
- Free‑form markdown, but structured sections encouraged.
- If artifact (code, model, config), attach as separate file.
```

### **Routing Rules**  
- **Broadcast:** All agents read, but only addressed agent(s) must act.  
- **Direct:** Bottle placed in recipient’s `inbox/` subdirectory (to be implemented).  
- **Alert:** High‑priority messages trigger Sentinel health checks.  

### **Validation**  
- Each bottle must reference at least one real file from `fleet‑knowledge/`.  
- No absolute claims (“always works”, “never fails”).  
- Output must be factual and specific.
