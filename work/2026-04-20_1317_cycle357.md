## 2. Communication Protocol Gaps & P1 Channel Identification

**P0 (Negative Space / What NOT to do):**
*   Do not implement a complex, real-time messaging system. It adds overhead and central point of failure.
*   Do not allow bottles to become stale (e.g., unread for >48 hours) without a cleanup mechanism.

**P1 (Safe Channels / Where we CAN be):**
1.  **Maintain the Bottle System:** It works. The current directory-based broadcast is a valid, simple channel.
2.  **Introduce a Read/Unread Signal:** A companion `.lock` or `.read` file created when an agent processes a bottle.
3.  **Create a Herald Digest:** A periodic (e.g., daily) summary bottle synthesized from all incoming messages, providing a unified fleet status.

**P2 (Optimization):**
*   Add metadata headers to bottles (e.g., `## METADATA: To: Oracle1, Priority: High, Topic: Training-Data`).
*   Develop a simple CLI tool (`bottle-post`, `bottle-read`) to standardize interaction.
