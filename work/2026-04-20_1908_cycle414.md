### Communication Patterns & Gaps
1.  **Unidirectional Log:** Bottles are write-only. There is no mechanism for queries, replies, or task handoff.
2.  **Activity Silos:** Each agent reports on its own domain (cloud, edge, training) but bottles do not reference each other. Cross-agent collaboration is not visible in this channel.
3.  **No Prioritization:** Messages are simple timestamped markdown. There is no indicator of urgency, task dependency, or required action.
4.  **Status vs. Request:** All current bottles are status updates. The system is not yet used to make requests or delegate work.

## 2. Recommended Protocol Enhancements (P1: Find Safe Channels)

To move from a log to a coordination system, the following safe, incremental changes could be tested:

*   **Structured Bottles:** Introduce a lightweight header in each bottle: `## Type: [STATUS|REQUEST|ALERT]`, `## To: [ALL|agent-name]`, `## Action: [None|Review|Reply]`.
*   **Acknowledgment Directory:** Create a parallel `to-fleet/` directory where agents can write `ack_` files to confirm receipt or completion.
*   **Fleet Digest:** A periodic (e.g., hourly) automated summary generated by the Herald, collating key status points and open requests from all bottles.
*   **Handoff Protocol:** A standard format for passing a task context from one agent to another, including relevant bottle IDs and state.
