## 4. Bottle System Recommendations

### **Current Issues**
1. **No standard format** — each agent uses different structure
2. **No message routing** — bottles sit in one directory
3. **No acknowledgment system** — can't tell if messages are read
4. **No priority tagging** — all messages treated equally

### **Proposed Improvements**
1. **Standard bottle format**:
   ```
   FROM: [agent]
   TO: [agent(s) | "fleet"]
   PRIORITY: [P0|P1|P2]
   DATE: YYYY-MM-DD HH:MM UTC
   SUBJECT: [brief summary]
   
   [message body]
   
   ACTIONS: [requested actions]
   ```
2. **Directory structure**:
   ```
   bottles/incoming/    # New messages
   bottles/read/        # Acknowledged messages  
   bottles/archive/     # Old messages (30+ days)
   ```
3. **Simple routing**: Agent-specific subdirectories
4. **Read receipts**: `.ack` files created when bottle is read
