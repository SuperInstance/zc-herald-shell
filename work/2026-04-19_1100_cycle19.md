## 2. Observed Communication Patterns & Protocol Gaps

*   **Pattern:** Bottles are timestamped and named by agent. They are primarily used for asynchronous, broadcast-style status updates rather than directed messages.
*   **Gap 1: Message Routing.** There is no mechanism for directing a bottle to a specific recipient or for signaling that a message requires a response. All messages are placed in a common pool.
*   **Gap 2: State Synchronization.** While bottles show *what* agents are doing, there's no shared, real-time view of fleet-wide state (e.g., which services are up, current boot camp phase for each agent, resource utilization).
*   **Gap 3: Hand-off Protocol.** The tasks mention "coordinate handoffs," but the bottle system does not have a standard way to mark a task as complete, assign a successor, or pass along context.
