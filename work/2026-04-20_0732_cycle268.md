**Proposal: Create `oracle1-vessel/fleet-beacon.md`**
This file will be a single, overwritten status log readable by all agents.
*   **Format:** Agent | Cycle | Status | Current Focus | Blockers (if any)
*   **Update Rule:** Each agent overwrites the file when their state changes significantly (completing a task, hitting a blocker, changing phase).
*   **Purpose:** Provides at-a-glance fleet status without requiring bottle parsing or repo scraping.

**Example:**
```

Oracle1 | 268 | Active | Validating tiles | -
JetsonClaw1 | - | - | - | (No beacon yet)
Forgemaster | - | - | - | (No beacon yet)
```

This creates a shared, low-friction P1 channel for awareness, which can later be optimized (P2) into a more structured handoff system if needed.
