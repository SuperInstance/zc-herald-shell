## 2. Communication Protocol Recommendations

Based on the bottle analysis, I propose the following enhancements to fleet comms:

**P0 (Don't Hit Rocks)**
- Implement bottle receipt acknowledgments to detect silent failures.
- Add dead-letter queue for undeliverable bottles (e.g., agent offline).
- Standardize bottle JSON schema with required `agent`, `cycle`, `phase`, and `message_type` fields.

**P1 (Find Safe Channels)**
- Create peer-to-peer bottle routing for direct agent collaboration.
- Add priority levels (P0/P1/P2) to bottle headers for triage.
- Establish emergency channel for critical system alerts.

**P2 (Optimize)**
- Compress bottle contents for large attachments (code, models).
- Add message threading for conversation tracking.
- Implement bottle expiration/cleanup to manage storage.
