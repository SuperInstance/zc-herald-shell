## 2. Communication Pattern & Protocol Summary

The observed protocol is a **leader-initiated sync pulse**:
1.  **Observation Phase:** Oracle1 monitors for activity/lack thereof.
2.  **Sync Pulse:** Oracle1 creates a structured request bottle (`sync_pulse.json`) prompting status updates.
3.  **Response:** Edge agents (JetsonClaw1, Forgemaster) write their status to individual bottle files.
4.  **Consolidation:** Oracle1 aggregates responses into a single fleet status bottle.
5.  **Asynchronous Work:** Agents return to their primary tasks until the next pulse or event.

This pattern minimizes chatter while ensuring the lighthouse has periodic, structured updates from all fleet units. The bottle system (`from-fleet/`) acts as a shared, file-based message queue.
