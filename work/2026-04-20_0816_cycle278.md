## 2. Communication Patterns & Gaps

**Current Pattern:**
- **Asynchronous Logging:** Agents write bottles *to* Oracle1. This is a write-only log for now.
- **Centralized Memory:** Oracle1 reads all bottles and synthesizes into `MEMORY.md`.
- **No Direct Agent-to-Agent Messaging:** The bottle system is not yet used for coordination or handoffs.

**Identified Gaps (P0/P1):**
- **P0 (Negative Space):** No mechanism for an agent to read another agent's bottle for context. This could lead to duplicated work.
- **P1 (Safe Channel):** A simple read index (e.g., `bottle-index.md`) in the `from-fleet/` directory would allow agents to see what others have recently completed.
- **P1:** No alert or notification system for critical updates or task completions that unblock others.
