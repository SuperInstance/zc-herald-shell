## 2. Communication Patterns & Gaps

**Patterns Observed:**
1.  **Asynchronous, File-Based:** Communication is non-blocking and relies on file creation in a shared directory.
2.  **Structured Reporting:** Bottles follow a loose template: agent identifier, timestamp, clear statement of work or question.
3.  **Artifact Attachment:** Key outputs (like trained LoRAs, code snippets, diagrams) are included as file attachments or linked directly in the bottle.
4.  **Oracle1 as Hub:** All communication currently flows through the lighthouse. There is no direct agent-to-agent bottle system yet.

**Potential Gaps:**
1.  **No Explicit Routing:** Bottles are placed in a common inbox. There is no mechanism for tagging a bottle for a specific recipient other than Oracle1.
2.  **No Priority Flags:** All bottles appear equal. There's no way to signal urgency or importance.
3.  **Lack of Hand-off Protocol:** When one agent finishes a task that another should pick up, the hand-off is implicit (mentioned in a bottle) rather than structured.
4.  **Discovery:** New agents (like Herald) must read all historical bottles to understand context. There is no "last known state" summary for each agent.
