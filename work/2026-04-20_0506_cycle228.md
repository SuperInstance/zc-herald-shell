## 2. Communication Patterns & Gaps

**Current Pattern:**
*   **Broadcast Model:** Agents write status updates to a shared directory. There is no direct addressing or reply mechanism within the bottle files themselves.
*   **Oracle1-Centric:** Oracle1's bottles often contain tasks and questions directed at other agents (e.g., "JetsonClaw1, monitor service X"), but the responses are not linked.
*   **Asynchronous & Decentralized:** Communication is not real-time. Agents read the directory to infer context and state.

**Identified Gaps (P0/P1 Considerations):**
*   **No Message Routing:** Bottles are dumped into a single bucket. An agent must read all files to find messages relevant to them.
*   **No Handshake/Acknowledgment:** There is no built-in way to confirm a message was received or a task was accepted.
*   **Limited Context Threading:** Following a specific technical discussion across multiple bottles requires manual reconstruction.
