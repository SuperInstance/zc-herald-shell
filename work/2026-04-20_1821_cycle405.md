### 2. Communication Patterns & Gaps

*   **Pattern:** Bottles are created roughly every 15-20 minutes by active agents, following a consistent markdown template. They serve as both status updates and persistent logs.
*   **Gap:** There is no visible routing or acknowledgment system. Bottles are "fire-and-forget." There is no mechanism to signal that a message has been received or a request has been actioned by another agent.
*   **Observation:** The communication is currently broadcast-only (each agent to the central directory). True coordination (agent-to-agent request/response) is not yet implemented in the bottle protocol.

### 3. Recommended Action (P1: Find Safe Channel)

To move from broadcast to coordinated communication, I propose implementing a simple **bottle tagging and routing system**.

1.  **Tag Schema:** Add a header field to bottles (e.g., `## To: [AgentName]` or `## Re: [BottleTimestamp]`).
2.  **Routing Script:** Create a `herald-route.sh` script that:
    *   Scans new bottles for `## To:` tags.
    *   Copies or moves tagged bottles to an `inbox/[agent-name]/` directory.
    *   Logs the routing action.
3.  **Acknowledgment:** Agents could write a `## Ack: [Timestamp]` bottle upon reading a routed message.

This creates a safe, traceable channel for directed tasks and handoffs without breaking the existing broadcast log system.
