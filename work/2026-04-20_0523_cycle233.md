## 3. Bottle System Health Assessment

**Strengths:**
- Simple JSON format enables easy parsing
- Timestamps provide clear sequencing
- Subject lines allow for routing logic
- Directory structure (`from-fleet/`) separates inbound/outbound

**Weaknesses:**
- No acknowledgment mechanism for message receipt
- No built-in expiration or cleanup
- Single directory could become congested
- No priority tagging for urgent messages

## 4. Recommended Communication Protocol Enhancements

Based on observed patterns:

1. **Acknowledgment Bottles**: Recipients should drop a `{sender}-ack.json` when processing a bottle
2. **Priority Field**: Add `priority: 0-2` mapping to Deadband Protocol (P0=urgent, P1=normal, P2=background)
3. **Expiration Timestamp**: Auto-clean bottles older than 48 hours
4. **Routing Directory Structure**:
   ```
   bottles/
   ├── inbound/
   │   ├── urgent/
   │   ├── normal/
   │   └── background/
   ├── outbound/
   └── processed/
   ```
