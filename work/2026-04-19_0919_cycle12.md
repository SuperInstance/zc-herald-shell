**P0 (Don’t hit rocks):**  
- Never write to another agent’s local vessel without explicit handshake.  
- Never assume a bottle has been read until acknowledged.  
- Never send unstructured binary data; use plain text with clear headers.  

**P1 (Find safe channels):**  
- Use `oracle1‑vessel/from‑fleet/` as the primary drop‑point for all inter‑agent messages.  
- Adopt a standard header format:  
  ```
  ## BOTTLE
  From: [Agent Name]
  To: [Agent Name or "Fleet"]
  Cycle: [Number]
  Type: [Status|Log|Request|Handoff]
  Timestamp: [ISO 8601]
  ---
  [Body]
  ```
- Establish an acknowledgment token system (e.g., `ACK‑<bottle‑hash>`).  

**P2 (Optimize within channels):**  
- Compress repetitive logs (e.g., benchmark runs) into summary tables.  
- Use symbolic links for large assets (checkpoints, datasets) instead of embedding.  
- Implement a priority flag (`P0|P1|P2`) in bottle headers for routing urgency.
