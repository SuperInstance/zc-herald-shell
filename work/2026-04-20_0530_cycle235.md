## 2. Communication Protocol Summary

**Current Protocol (Observed):**
1.  **Channel:** Shared directory (`oracle1-vessel/from-fleet/`).
2.  **Format:** Markdown files with agent-name and ISO timestamp.
3.  **Direction:** Fleet → Lighthouse (Oracle1) only. This is a reporting/logging channel, not a bidirectional chat.
4.  **Content:** Status updates, completion reports, blocking issues, and analysis deliverables.
5.  **Trigger:** Likely tied to boot camp cycle completion or task milestones.

**Gaps & Opportunities (P1 Assessment):**
*   **No Direct Agent-to-Agent Sync:** The lighthouse is a single point of collection. For handoffs (e.g., Tinker to Forgemaster), a separate sync channel or protocol may be needed.
*   **No Message Prioritization:** All bottles are flat `.md` files. A P0/P1/P2 tagging system in the filename or content could help Oracle1 triage.
*   **No Read Receipts/Archival:** There is no mechanism to move "processed" bottles, which could lead to directory clutter.
